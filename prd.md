# 📦 Project Requirements Document (PRD) – Lösen SaaS-1

## 🎯 Vision
Build **Lösen** – a sales‑intelligence SaaS that aggregates contacts, tracks interactions, colours status, monitors company news, and provides a weekly AI‑generated outreach list.

## 👥 Primary Users
- **Salespeople** that manage many prospects across Excel, OneNote, email & LinkedIn.
- **Account managers** needing a single dashboard to see last touch‑point, next activity and colour‑coded status.

## 📋 Core Features (MVP)
1. **Contact Dashboard**
   - List all contacts with: name, email, phone, LinkedIn, company.
   - Show **lastTouchpoint** and **nextActivity** timestamps.
   - Colour‑code rows:
     - 🟢 Green – activity scheduled.
     - 🟡 Yellow – recent contact (<4 weeks) but no scheduled activity.
     - 🔴 Red – no contact >4 weeks.
2. **Company Intelligence**
   - Pull news via **NewsAPI** (or Google News RSS).
   - Detect keywords: `layoffs`, `varsel`, `order`, `new contract`, `marketing campaign`.
   - Push **real‑time alerts** (Telegram, email).
3. **Weekly AI Assistant**
   - Run a **weekly job** that scans contacts & company news.
   - Generate a list of 5‑10 high‑priority prospects.
   - Provide a **reason** and a **pitch snippet**.
4. **Authentication & Multi‑User**
   - Users sign‑in with email/password (Supabase Auth).
   - Row‑Level Security (RLS) isolates each user’s data.
5. **CRUD API** (generated by Supabase) plus custom edge‑functions for AI.

## 🛠️ Tech Stack
| Layer | Technology |
|-------|------------|
| Front‑end | **Next.js** (React) + TailwindCSS |
| Back‑end | **Supabase** (PostgreSQL) – auto‑generated REST + Realtime |
| AI | **OpenAI GPT‑4** (prompt in Edge Function) |
| News | **NewsAPI.org** (free tier) |
| Notifications | **Telegram Bot API** / **SendGrid** |
| Auth | Supabase Auth |
| Deploy | **Vercel** (frontend) + Supabase Cloud (DB) |

## 📦 Database Schema (Prisma‑style)
```prisma
model User {
  id        String   @id @default(uuid())
  email     String   @unique
  password  String   // hashed by Supabase Auth
  contacts  Contact[]
}

model Contact {
  id               String   @id @default(uuid())
  userId           String   @relation(fields: [userId], references: [id])
  name             String
  email            String?
  phone            String?
  linkedinUrl      String?
  companyId        String?
  lastTouchpoint   DateTime?
  nextActivity     DateTime?
  status           String   @default("yellow") // green|yellow|red
  notes            String?
  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt
}

model Company {
  id           String   @id @default(uuid())
  name         String
  industry     String?
  website      String?
  newsKeywords String[]
  newsItems    NewsItem[]
}

model Activity {
  id          String   @id @default(uuid())
  contactId   String   @relation(fields: [contactId], references: [id])
  type        String   // call, meeting, email, linkedin
  notes       String?
  timestamp   DateTime @default(now())
  outcome     String?
}

model NewsItem {
  id          String   @id @default(uuid())
  companyId   String   @relation(fields: [companyId], references: [id])
  title       String
  url         String
  source      String
  newsType    String   // layoff, order, media, recruitment
  publishedAt DateTime
}
```

## 🚀 Deployment Steps (high‑level)
1. **Create Supabase project** – name `saas-1` (or `Lösen`).
2. **Apply the schema** – via Supabase SQL editor or `supabase db push`.
3. **Enable RLS** and create policies so a user can only read/write their own contacts.
4. **Add Auth** – email/password sign‑up.
5. **Deploy Next.js** to Vercel – connect to Supabase URL & anon‑key.
6. **Configure env vars**:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY` (secure, for server‑side edge functions).
7. **Add OpenAI secret** (`OPENAI_API_KEY`).
8. **Add NewsAPI key** (`NEWSAPI_KEY`).
9. **Add Telegram Bot token** (`TELEGRAM_BOT_TOKEN`).

## 📋 Acceptance Criteria (MVP)
- ✅ Users can register / log‑in.
- ✅ Contact list shows colour‑coded rows based on the status logic.
- ✅ Adding / editing a contact updates the status automatically.
- ✅ Company news alerts appear in a side‑panel and send a Telegram push.
- ✅ Weekly AI job runs, stores a list of suggestions, and presents them on a “Weekly Leads” page.
- ✅ All endpoints are protected by RLS – no cross‑user data leakage.
- ✅ Deploy to Vercel and Supabase with no manual DB migrations needed after push.

---
*All secrets (passwords, API keys) must be stored only in environment variables – never hard‑coded.*
